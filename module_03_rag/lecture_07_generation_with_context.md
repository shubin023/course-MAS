# Раздел 7: Генерация с контекстом — Искусство ответа на основе источников

## Введение: Последняя миля RAG

Пройден долгий путь: научились превращать документы в векторы, хранить их в специализированных базах данных, находить релевантные фрагменты, переранжировать результаты. Всё это — подготовка к кульминационному моменту: генерации ответа.

Представьте себе блестящего аналитика, которому вы передали папку с идеально подобранными документами по его запросу. Документы релевантны, структурированы, отсортированы по важности. Но успех по-прежнему зависит от того, как аналитик прочитает эти документы, синтезирует информацию и сформулирует ответ. Он может упустить ключевую деталь, неправильно интерпретировать данные, или "выдумать" то, чего в документах нет.

Генерация ответа в RAG — это тот самый момент, когда все наши инженерные усилия должны превратиться в полезный ответ пользователю. И здесь критически важны две вещи: как мы инструктируем модель использовать контекст (промпт-инжиниринг) и как обеспечиваем доверие через цитирование источников.

## Часть 1: Анатомия RAG-промпта

### Базовая структура

Эффективный RAG-промпт состоит из нескольких компонентов:

```
[Системная инструкция]
Ты — помощник по корпоративным политикам. Твоя задача — отвечать на вопросы
сотрудников, используя ТОЛЬКО информацию из предоставленного контекста.

[Правила поведения]
- Если ответа нет в контексте, честно скажи об этом
- Не выдумывай информацию
- Цитируй источники

[Контекст]
---
Документ 1: [текст]
Источник: HR_Policy_2024.pdf, стр. 15
---
Документ 2: [текст]
Источник: Vacation_Procedure.docx
---

[Вопрос пользователя]
Сколько дней отпуска положено сотруднику со стажем 5 лет?

[Формат ответа]
Ответь на вопрос, указывая источники в формате [Источник: название документа].
```

### Ключевые элементы промпта

**1. Системная инструкция (System Prompt)**

Определяет роль и общее поведение модели. Важно задать:
- Роль ("эксперт по HR-политикам")
- Ограничение источников ("только из контекста")
- Тон ("профессиональный, но дружелюбный")

```python
SYSTEM_PROMPT = """Ты — ассистент корпоративной базы знаний компании "АльфаТех".
Твоя задача — помогать сотрудникам находить информацию в корпоративных документах.

Принципы работы:
1. Используй ТОЛЬКО информацию из предоставленного контекста
2. Если контекст не содержит ответа — честно сообщи об этом
3. Не делай предположений и не добавляй информацию от себя
4. Указывай источники при цитировании"""
```

**2. Инструкция по работе с контекстом**

Явно объясняем модели, что делать с контекстом:

```python
CONTEXT_INSTRUCTION = """
Ниже приведены фрагменты документов, найденные по запросу пользователя.
Каждый фрагмент содержит текст и метаданные об источнике.

Инструкции:
- Внимательно прочитай все фрагменты перед ответом
- Синтезируй информацию из нескольких источников, если это необходимо
- При противоречиях между источниками — укажи на это
- Если информации недостаточно для полного ответа — укажи, что ещё нужно уточнить
"""
```

**3. Форматирование контекста**

Как структурировать найденные документы:

```python
def format_context(documents):
    formatted = []
    for i, doc in enumerate(documents, 1):
        source = doc.metadata.get("source", "Неизвестный источник")
        page = doc.metadata.get("page", "")
        page_str = f", стр. {page}" if page else ""

        formatted.append(f"""
--- Документ {i} ---
Источник: {source}{page_str}

{doc.page_content}
""")
    return "\n".join(formatted)
```

**4. Формат выходных данных**

Можем требовать структурированный ответ:

```python
OUTPUT_FORMAT = """
Ответ должен содержать:
1. Краткий ответ на вопрос (1-2 предложения)
2. Детальное объяснение с цитатами из источников
3. Список использованных источников

Формат цитирования: [Источник: название_документа]
"""
```

## Часть 2: Стратегии промптинга для RAG

### Стратегия 1: "Только контекст" (Strict RAG)

Максимально ограничиваем модель использованием только предоставленного контекста.

```python
STRICT_RAG_PROMPT = """
Ты — система ответов на вопросы с ЖЁСТКИМ ограничением.
Ты МОЖЕШЬ использовать ТОЛЬКО информацию из контекста ниже.
Ты НЕ МОЖЕШЬ использовать свои знания.

Если ответа НЕТ в контексте, ответь ТОЧНО так:
"К сожалению, в предоставленных документах нет информации по этому вопросу."

Контекст:
{context}

Вопрос: {question}

Ответ:
"""
```

Плюсы: минимум галлюцинаций, максимальное соответствие источникам.
Минусы: не сможет объяснить терминологию, ответ может быть "сухим".

### Стратегия 2: "Контекст + базовые знания" (Augmented RAG)

Разрешаем модели использовать общие знания для объяснения, но факты — только из контекста.

```python
AUGMENTED_RAG_PROMPT = """
Ты — эксперт-консультант.

При ответе на вопрос:
- ФАКТЫ и ДАННЫЕ бери ТОЛЬКО из контекста ниже
- Для ОБЪЯСНЕНИЙ и ТЕРМИНОВ можешь использовать общие знания
- Чётко разделяй, что из контекста, а что — общее знание

Контекст:
{context}

Вопрос: {question}

Ответ:
"""
```

Плюсы: более понятные ответы, может объяснить сложные термины.
Минусы: риск смешения фактов из контекста и общих знаний.

### Стратегия 3: "Аналитический RAG" (Synthesis RAG)

Для вопросов, требующих анализа и синтеза информации.

```python
SYNTHESIS_RAG_PROMPT = """
Ты — аналитик, которому нужно ответить на сложный вопрос.

Подход к ответу:
1. Сначала определи, какие аспекты вопроса затрагиваются в контексте
2. Для каждого аспекта — найди релевантную информацию
3. Синтезируй ответ, объединяя информацию из разных источников
4. Укажи, если какие-то аспекты вопроса не покрыты контекстом

Контекст:
{context}

Вопрос: {question}

Анализ и ответ:
"""
```

### Стратегия 4: "Диалоговый RAG" (Conversational RAG)

Для многоходовых диалогов с сохранением истории.


## Часть 3: Цитирование источников

### Почему цитирование критично

Цитирование решает несколько проблем:

**1. Доверие:** Пользователь может проверить информацию.

**2. Отладка:** Разработчик видит, откуда взята информация.

**3. Детекция галлюцинаций:** Если модель "цитирует" несуществующий источник — это сигнал проблемы.

**4. Юридическая защита:** В регулируемых отраслях нужно показывать источники.

### Inline цитирование

Источники указываются прямо в тексте ответа:

```
Сотрудникам со стажем более 3 лет положено 28 календарных дней
отпуска [Источник: Политика_отпусков.pdf], при этом минимальная
непрерывная часть должна составлять 14 дней [Источник: ТК_РФ_справка.docx].
```

**Реализация:**

```python
INLINE_CITATION_PROMPT = """
При ответе используй inline-цитаты в формате [n], где n — номер документа.
После ответа приведи список источников.

Документы:
[1] {doc1_source}: {doc1_content}
[2] {doc2_source}: {doc2_content}

Вопрос: {question}

Ответ с цитатами:
"""
```

### Структурированное цитирование

Ответ и источники разделены:

```json
{
  "answer": "Сотрудникам со стажем более 3 лет положено 28 календарных дней отпуска.",
  "citations": [
    {
      "source": "Политика_отпусков.pdf",
      "page": 5,
      "quote": "Работникам с непрерывным стажем от 3 лет предоставляется отпуск продолжительностью 28 к.д."
    }
  ],
  "confidence": "high"
}
```

**Реализация с Pydantic:**

```python
from pydantic import BaseModel, Field
from typing import List, Optional
from langchain_openai import ChatOpenAI

class Citation(BaseModel):
    source: str = Field(description="Название документа-источника")
    page: Optional[int] = Field(description="Номер страницы, если есть")
    quote: str = Field(description="Дословная цитата из документа")

class RAGResponse(BaseModel):
    answer: str = Field(description="Ответ на вопрос пользователя")
    citations: List[Citation] = Field(description="Список цитат с источниками")
    confidence: str = Field(description="Уверенность: high/medium/low")
    missing_info: Optional[str] = Field(description="Что не найдено в документах")

llm = ChatOpenAI(model="gpt-5").with_structured_output(RAGResponse)

response = llm.invoke(prompt)
print(response.answer)
for cite in response.citations:
    print(f"  - {cite.source}: \"{cite.quote}\"")
```

### Верификация цитат

Модель может "придумать" цитату, которой нет в контексте. Верификация:

```python
def verify_citations(answer, citations, documents):
    """Проверяет, что цитаты действительно есть в документах"""
    verified = []
    hallucinated = []

    for citation in citations:
        found = False
        for doc in documents:
            if citation.quote.lower() in doc.page_content.lower():
                found = True
                break

        if found:
            verified.append(citation)
        else:
            hallucinated.append(citation)

    return {
        "verified_citations": verified,
        "hallucinated_citations": hallucinated,
        "citation_accuracy": len(verified) / len(citations) if citations else 1.0
    }
```

```python
def format_context_with_dates(documents):
    """Добавляем информацию о давности документа"""
    from datetime import datetime

    formatted = []
    for doc in documents:
        doc_date = doc.metadata.get("date")
        if doc_date:
            days_old = (datetime.now() - doc_date).days
            freshness = ""
            if days_old > 365:
                freshness = "⚠️ Документ старше года"
            elif days_old > 180:
                freshness = "ℹ️ Документу более 6 месяцев"

            formatted.append(f"""
{doc.page_content}
[Источник: {doc.metadata['source']}, дата: {doc_date.strftime('%d.%m.%Y')} {freshness}]
""")
    return "\n".join(formatted)
```

### Self-Reflection: проверка собственного ответа

```python
def generate_with_reflection(query, context, llm):
    # Шаг 1: Генерация ответа
    initial_response = llm.invoke(f"""
    Контекст: {context}
    Вопрос: {query}
    Ответ:
    """)

    # Шаг 2: Рефлексия
    reflection = llm.invoke(f"""
    Проверь этот ответ на соответствие контексту.

    Контекст: {context}
    Вопрос: {query}
    Ответ: {initial_response}

    Анализ:
    1. Все ли факты в ответе подтверждаются контекстом? (Да/Нет + детали)
    2. Есть ли в ответе информация, которой НЕТ в контексте? (Да/Нет + детали)
    3. Пропущена ли важная информация из контекста? (Да/Нет + детали)

    Итог: ответ корректен / требует исправления
    """)

    # Шаг 3: Исправление при необходимости
    if "требует исправления" in reflection.lower():
        final_response = llm.invoke(f"""
        Исправь ответ на основе рефлексии.

        Контекст: {context}
        Исходный ответ: {initial_response}
        Замечания: {reflection}

        Исправленный ответ:
        """)
        return final_response

    return initial_response
```

```python
def compress_context(documents, query, llm, max_length=2000):
    """Извлекает только релевантные части из документов"""
    compressed_docs = []

    for doc in documents:
        compressed = llm.invoke(f"""
        Извлеки из текста ТОЛЬКО те части, которые релевантны вопросу.
        Сохрани точные формулировки — не перефразируй.

        Текст:
        {doc.page_content}

        Вопрос:
        {query}

        Релевантные части (или "Нет релевантной информации"):
        """)

        if "нет релевантной информации" not in compressed.lower():
            compressed_docs.append({
                "content": compressed,
                "source": doc.metadata.get("source")
            })

    return compressed_docs
```

## Часть 6: Полный RAG Pipeline

### Собираем всё вместе

```python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.runnables import RunnablePassthrough
from langchain_core.output_parsers import StrOutputParser

# Компоненты
llm = ChatOpenAI(model="gpt-5", temperature=0)
retriever = create_reranking_retriever(vectorstore)

# Промпт
RAG_PROMPT = ChatPromptTemplate.from_template("""
<system>
Ты — ассистент корпоративной базы знаний.
Отвечай ТОЛЬКО на основе предоставленного контекста.
Если информации нет — честно скажи об этом.
Всегда указывай источники в формате [Источник: название].
</system>

<context>
{context}
</context>

<question>
{question}
</question>

<answer>
""")

# Форматирование контекста
def format_docs(docs):
    formatted = []
    for i, doc in enumerate(docs, 1):
        source = doc.metadata.get("source", "Неизвестно")
        formatted.append(f"[Документ {i}]\nИсточник: {source}\n\n{doc.page_content}")
    return "\n\n---\n\n".join(formatted)

# RAG Chain
rag_chain = (
    {
        "context": retriever | format_docs,
        "question": RunnablePassthrough()
    }
    | RAG_PROMPT
    | llm
    | StrOutputParser()
)

# Использование
answer = rag_chain.invoke("Какие документы нужны для оформления отпуска?")
print(answer)
```

```python
from langchain_community.callbacks import get_openai_callback
import time

def invoke_with_metadata(chain, query):
    """Вызов с сбором метаданных"""
    start_time = time.time()

    with get_openai_callback() as cb:
        result = chain.invoke(query)

    metadata = {
        "query": query,
        "answer": result,
        "latency_ms": (time.time() - start_time) * 1000,
        "tokens_used": cb.total_tokens,
        "cost_usd": cb.total_cost
    }

    return result, metadata
```

## Часть 7: Практические рекомендации

### Чек-лист качественного RAG-промпта

1. **Чёткая роль:** Кто модель? Какой тон?
2. **Ограничения:** Что можно, что нельзя?
3. **Формат контекста:** Как структурированы документы?
4. **Формат ответа:** Ожидаемая структура выхода
5. **Обработка краевых случаев:** Нет ответа, противоречия, неясность

### Типичные ошибки

**Ошибка 1: Слишком мягкие ограничения**

Плохо: "Используй контекст для ответа"
Хорошо: "Используй ТОЛЬКО контекст. Если информации нет — скажи прямо."

**Ошибка 2: Нет инструкции по цитированию**

Без явной инструкции модель может не цитировать или цитировать неправильно.

**Ошибка 3: Перегруженный контекст**

Слишком много документов = модель "теряется". Используйте reranking и компрессию.

**Ошибка 4: Игнорирование temperature**

Для RAG рекомендуется temperature=0 или близко к нему — максимальная "фактичность".

### Метрики качества генерации

**Faithfulness:** Соответствует ли ответ контексту?

## Заключение: RAG как целостная система

Завершаем модуль RAG на финальном аккорде — генерации ответа. Но важно помнить: качество генерации определяется всеми предыдущими этапами.

**Garbage in — Garbage out:** Плохо подготовленные данные → плохой retrieval → плохой ответ.

**Цепочка качества:** Chunking → Embedding → Indexing → Retrieval → Reranking → Generation. Слабое звено определяет качество всей системы.

Ключевые идеи раздела:

**Промпт определяет поведение:** Чёткие инструкции, ограничения, формат — залог качественных ответов.

**Цитирование — не роскошь:** Для доверия, отладки, юридической защиты — источники обязательны.

**Особые случаи требуют обработки:** Нет ответа, противоречия, уточнения — всё должно быть предусмотрено.

**Верификация необходима:** Модель может галлюцинировать даже с контекстом. Проверяйте цитаты.

RAG — это не просто "подложить документы в промпт". Это сложная инженерная система, требующая внимания к каждому компоненту. Но когда все части работают вместе — результат впечатляет: интеллектуальный ассистент, который знает вашу базу знаний лучше любого сотрудника.

## Ключевые термины

| Термин | Определение |
|--------|-------------|
| **System Prompt** | Инструкция, определяющая роль и поведение модели |
| **Inline Citation** | Указание источника прямо в тексте ответа |
| **Structured Output** | Ответ в формализованном формате (JSON, Pydantic) |
| **Faithfulness** | Метрика соответствия ответа контексту |
| **Context Compression** | Извлечение только релевантных частей документов |
| **Self-Reflection** | Проверка моделью собственного ответа |
| **Temperature** | Параметр "креативности" модели (0 = детерминированно) |
| **Citation Verification** | Проверка наличия цитат в исходных документах |
